# STM32 OLED温度监控系统 v2.0

## 项目简介

这是一个基于STM32F103C8T6微控制器的温度监控系统，使用NTC热敏电阻进行温度采集，并通过SSD1306 OLED显示屏实时显示温度数据。

## 硬件配置

- **微控制器**: STM32F103C8T6 (72MHz)
- **温度传感器**: NTC热敏电阻 10kΩ @ 25°C (Beta=3950)
- **显示屏**: SSD1306 OLED (128x64像素)
- **ADC输入**: PA0 (ADC1_IN0)
- **I2C接口**: PB8 (SCL), PB9 (SDA) - 软件模拟

## 主要功能

### 1. 温度采集
- 32次采样平均值，提高测量精度
- 基于Beta参数模型的温度计算
- 自动温度范围验证 (-20°C ~ 100°C)
- 实时统计最小/最大温度值

### 2. OLED显示
- **启动画面**: 显示版本信息和欢迎界面
- **主显示页面**: 
  - 实时温度显示 (精确到0.1°C)
  - 温度条形图可视化
  - 系统状态指示
- **统计页面**: 
  - 历史最低/最高温度
  - ADC原始值
  - 采样次数统计
- **报警界面**: 温度异常时显示警告信息

### 3. 温度报警
- 高温报警阈值: 80°C
- 低温报警阈值: 0°C
- 可配置的报警开关
- 自动报警界面切换

### 4. 系统特性
- 模块化设计，易于维护和扩展
- 完善的错误处理机制
- 500ms温度更新间隔
- 双页面自动切换显示

## 配置参数

### 温度传感器参数
```c
#define NTC_BETA 3950.0f         // NTC Beta系数
#define NTC_R0 10000.0f          // 25°C时的阻值
#define R_PULLUP 10000.0f        // 上拉电阻阻值
#define VREF 3.3f                // 参考电压
```

### ADC采样参数
```c
#define ADC_SAMPLES 32           // 采样次数
#define ADC_SAMPLE_DELAY_MS 1    // 采样间隔
```

### 温度报警参数
```c
#define TEMP_MIN -20.0f          // 最小温度
#define TEMP_MAX 100.0f          // 最大温度
#define TEMP_WARNING_HIGH 80.0f  // 高温报警阈值
#define TEMP_WARNING_LOW 0.0f    // 低温报警阈值
```

## 使用方法

1. **编译项目**: 使用STM32CubeIDE打开项目并编译
2. **烧录程序**: 通过ST-Link或其他调试器烧录到STM32
3. **连接硬件**: 
   - NTC热敏电阻连接到PA0
   - OLED连接到PB8(SCL)和PB9(SDA)
4. **上电运行**: 系统自动启动并显示温度数据

## 显示页面说明

### 页面1 - 主显示
- 顶部: "TEMP MONITOR" 标题
- 中部: 实时温度值 (格式: T:XX.X°C)
- 下部: 温度条形图
- 底部: 系统状态 (OK/WARN/ERR)

### 页面2 - 统计信息
- 顶部: "STATISTICS" 标题
- 中部: 最小/最大温度值
- 下部: ADC原始值和采样次数

### 报警页面
- 顶部: "!! WARNING !!" 警告信息
- 中部: 当前温度值
- 下部: "HIGH TEMP!" 或 "LOW TEMP!" 报警原因

## 技术特点

1. **高精度采样**: 32次采样平均，有效减少噪声
2. **模块化设计**: 功能模块分离，便于维护和扩展
3. **实时显示**: 500ms更新间隔，响应迅速
4. **智能报警**: 自动温度异常检测和报警
5. **数据统计**: 自动记录温度极值和采样次数
6. **可视化**: 温度条形图直观显示温度水平
