# 快速开始指南

## 硬件准备

### 必需硬件
- STM32F103C8T6开发板
- SSD1306 OLED显示屏 (128x64)
- NTC热敏电阻 (10kΩ @ 25°C)
- 10kΩ电阻 (作为上拉电阻)
- 面包板和连接线

### 硬件连接

#### NTC温度传感器连接
```
NTC热敏电阻 ────┬──── PA0 (ADC1_IN0)
                 │
                10kΩ 上拉电阻
                 │
                3.3V
```

#### OLED显示屏连接 (软件I2C)
```
OLED VCC  ──── 3.3V
OLED GND  ──── GND
OLED SCL  ──── PB8
OLED SDA  ──── PB9
```

## 软件准备

### 开发环境
- STM32CubeIDE (推荐)
- 或其他支持STM32的IDE

### 编译步骤
1. 打开STM32CubeIDE
2. 导入项目: File -> Import -> Existing STM32CubeIDE Project
3. 选择项目目录: `d:\stm32-oled-temp-monitor`
4. 点击Build按钮编译项目
5. 确认编译成功，无错误和警告

## 烧录程序

### 使用ST-Link
1. 连接ST-Link到STM32开发板
2. 在STM32CubeIDE中点击Run按钮
3. 程序将自动烧录并开始运行

### 使用其他调试器
1. 根据调试器类型进行相应配置
2. 选择正确的调试接口 (SWD/JTAG)
3. 烧录程序到开发板

## 运行测试

### 启动过程
1. 上电后，OLED显示启动画面 (2秒)
2. 进入温度监控主界面
3. 开始实时温度采集和显示

### 显示页面
- **页面1**: 实时温度 + 温度条形图 + 状态
- **页面2**: 统计信息 (最小/最大温度、ADC值、采样次数)
- **报警页面**: 温度异常时自动显示

### 页面切换
- 页面每400ms自动切换
- 温度异常时优先显示报警页面

## 参数配置

### 修改温度报警阈值
编辑 `Core/Inc/app_config.h`:
```c
#define TEMP_WARNING_HIGH 80.0f  // 修改高温报警阈值
#define TEMP_WARNING_LOW 0.0f    // 修改低温报警阈值
```

### 修改采样精度
编辑 `Core/Inc/app_config.h`:
```c
#define ADC_SAMPLES 32           // 修改采样次数 (1-100)
#define ADC_SAMPLE_DELAY_MS 1    // 修改采样间隔 (1-10ms)
```

### 修改NTC参数
编辑 `Core/Inc/app_config.h`:
```c
#define NTC_BETA 3950.0f         // 修改NTC Beta系数
#define NTC_R0 10000.0f          // 修改25°C时的阻值
#define R_PULLUP 10000.0f        // 修改上拉电阻阻值
```

### 修改更新速度
编辑 `Core/Inc/app_config.h`:
```c
#define TEMP_UPDATE_INTERVAL_MS 500  // 修改温度更新间隔
```

## 故障排除

### OLED不显示
1. 检查电源连接 (3.3V)
2. 检查I2C接线 (PB8, PB9)
3. 确认OLED地址为0x78
4. 检查上拉电阻

### 温度读数异常
1. 检查NTC连接
2. 确认上拉电阻正确
3. 验证NTC参数配置
4. 检查ADC输入电压范围

### 编译错误
1. 确认所有源文件已添加到项目
2. 检查头文件路径配置
3. 验证HAL库版本兼容性
4. 清理并重新编译项目

### 程序不运行
1. 检查电源供应
2. 验证晶振工作正常
3. 确认程序正确烧录
4. 检查启动模式配置
